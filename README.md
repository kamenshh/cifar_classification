# CIFAR Classification

Проект посвящен классификации цветных изображений из CIFAR-10. Это учебная задача компьютерного зрения: по маленькой картинке размером `32x32` модель должна понять, к какому из 10 классов она относится.

Классы в CIFAR-10: самолет, машина, птица, кот, олень, собака, лягушка, лошадь, корабль и грузовик. В проекте классы хранятся как числа от `0` до `9`.

## Идея проекта

Главная цель проекта - пройти полный цикл работы с изображениями:

- взять локально скачанный архив CIFAR-10;
- сохранить изображения в CSV-формате;
- посмотреть базовую информацию о данных;
- обучить сверточную модель;
- проверить качество модели через метрики.

## Данные

Исходный датасет лежит в архиве:

```text
cifar-10-python.tar.gz
```

Файл `dataset.py` читает этот архив напрямую, достает из него `data_batch_1`, `data_batch_2`, `data_batch_3`, `data_batch_4`, `data_batch_5` и сохраняет данные в `cifar10.csv`.

Формат CSV:

```text
label,pixel_0,pixel_1,pixel_2,...,pixel_3071
```

Первый столбец - номер класса. Остальные `3072` столбца - пиксели изображения, потому что `32 * 32 * 3 = 3072`.

В `education.py` данные обратно приводятся к форме:

```text
количество_картинок x 3 x 32 x 32
```

После этого значения пикселей нормализуются, чтобы модели было легче обучаться.

## Модель

В проекте используется простая CNN-модель. Она состоит из сверточных слоев, batch normalization, max pooling, dropout и двух полносвязных слоев в конце.

Сверточные слои ищут признаки на изображении: края, формы, пятна цвета и более сложные комбинации. Max pooling уменьшает размер признаков, dropout немного снижает переобучение, а последний слой выдает 10 чисел - по одному на каждый класс.

## Разделение данных

Датасет сохраняется в один CSV-файл. Потом в `education.py` он делится на обучающую и тестовую части через `train_test_split`.

Такой подход удобен для учебного проекта: все данные лежат в одном месте, а разделение происходит прямо перед обучением.

## Метрики

В проекте используются `accuracy` и `classification_report`.

### Accuracy

Accuracy показывает долю правильных ответов модели.

Например, если accuracy равна `0.70`, значит модель правильно классифицировала примерно 70% изображений из тестовой выборки.

### Precision

Precision показывает, насколько можно доверять предсказаниям конкретного класса.

Если модель часто говорит "кот", precision для класса "кот" показывает, какая часть этих ответов действительно была котами.

### Recall

Recall показывает, какую часть объектов конкретного класса модель смогла найти.

Например, если в тестовых данных было много собак, recall для класса "собака" показывает, сколько из них модель действительно распознала как собак.

### F1-score

F1-score объединяет precision и recall в одно число. Эта метрика полезна, когда нужно понять качество по каждому классу не только через точность угадывания, но и через пропуски.
